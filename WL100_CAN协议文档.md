# WL100 轮式底盘 CAN 通信标准协议文档 (修正版)

## 1. 协议概述

### 1.1 基本参数
- **通信方式**: CAN 2.0A 标准帧
- **波特率**: 500Kbps
- **数据长度**: 8字节
- **字节序**: 大端模式（High Byte First, 高字节在前）

### 1.2 消息ID分配
- **反馈消息**: 0x100 ~ 0x104（底盘 → 上位机）
- **控制消息**: 0x400 ~ 0x405（上位机 → 底盘）

---

## 2. 反馈消息（底盘 → 上位机）

### 2.1 0x100 - 底盘系统状态反馈
| 字节位置 | 字段内容                 | 数据类型 | 说明 |
|---------|-------------------------|---------|------|
| Byte[0] | 当前系统状态 (对应灯带状态) | Uint8 | 0x00:正常运行(绿灯), 0x01:低电量报警(红灯) |
| Byte[1] | 控制模式                 | Uint8 | 0x00:待机, 0x01:遥控器控制, 0x02:指令控制 |
| Byte[2] | 其它外设状态              | Uint8 | Bit0:急停, Bit1:避障, Bit2:接近开关, Bit3:安全触边 (0未触发/1已触发) |
| Byte[3] | 电池 SOC                | Uint8 | 电池电量百分比 0-100% |
| Byte[4] | 电池电压高八位           | Uint16| 结合Byte5，大端，解析单位 10mV |
| Byte[5] | 电池电压低八位           |        | |
| Byte[6] | 电池充放电电流高八位       | Int16 | 结合Byte7，大端，解析单位 10mA (充电为正，放电为负) |
| Byte[7] | 电池充放电电流低八位       |        | |

### 2.2 0x101 - 机器人四轮速度回馈
**发送周期**: 30ms
| 字节位置 | 字段内容 | 数据类型 | 说明 (数值单位: 0.001 m/s) |
|---------|------------------------|---------|------|
| Byte[0-1] | 1号电机转速 (左前轮) | Int16 | 高八位在前，低八位在后 (大端) |
| Byte[2-3] | 2号电机转速 (左后轮) | Int16 | 同上 |
| Byte[4-5] | 3号电机转速 (右前轮) | Int16 | 同上 |
| Byte[6-7] | 4号电机转速 (右后轮) | Int16 | 同上 |

### 2.3 0x102 - 机器人四轮角度回馈
**发送周期**: 30ms

| 字节位置 | 字段内容 | 数据类型 | 说明 (当前转向角度单位: 0.01°) |
|---------|------------------------|---------|------|
| Byte[0-1] | 5号电机转角 (左前转向) | Int16 | 大端 |
| Byte[2-3] | 6号电机转角 (左后转向) | Int16 | 同上 |
| Byte[4-5] | 7号电机转角 (右前转向) | Int16 | 同上 |
| Byte[6-7] | 8号电机转角 (右后转向) | Int16 | 同上 |

### 2.4 0x103 - 机器人故障定位回馈
**发送周期**: 1000ms
| 字节位置 | 字段内容 | 数据类型 | 说明 |
|---------|------------------------|---------|------|
| Byte[0] | 驱动故障标志位          | Uint8 | 0x00=驱动器正常，0x01=驱动器系统出现故障 |
| Byte[1] | 行走驱动器故障报警      | Uint8 | Bit0(1号)~Bit3(4号)，0正常/1故障 |
| Byte[2] | 转向驱动器故障报警      | Uint8 | Bit0(5号)~Bit3(8号)，0正常/1故障 |
| Byte[3] | 行走驱动器通讯断开报警   | Uint8 | Bit0(1号)~Bit3(4号)，0正常/1断开 |
| Byte[4] | 转向驱动器通讯断开报警   | Uint8 | Bit0(5号)~Bit3(8号)，0正常/1断开 |
| Byte[5-7]| 预留                   | - | 默认0 |

### 2.5 0x104 - 机器人运动状态回馈
**发送周期**: 20ms
| 字节位置 | 字段内容 | 数据类型 | 说明 |
|---------|------------------------|---------|------|
| Byte[0-1] | Vx (线速度 X方向) | Int16 | 大端，数值×1000上传, 前进正后退负 (0.001m/s) |
| Byte[2-3] | Vy (线速度 Y方向) | Int16 | 大端，数值×1000上传 (0.001m/s) |
| Byte[4-5] | Vz (角速度 Z方向) | Int16 | 大端，数值×1000上传, 左旋正右旋负 (0.001rad/s) |
| Byte[6] | 运动模式          | Uint8 | 0x01反向, 0x02正向, 0x04自旋, 0x08自主控制, 0x10驻车 |
| Byte[7] | 控制模式切换状态 | Uint8 | 0x00:切换完成, 0x01:切换过程中 |

## 3. 控制消息（上位机 → 底盘）

### 3.1 0x400 - 机器人控制模式下发
| 字节位置 | 字段内容 | 数据类型 | 说明 |
|---------|------------------------|---------|------|
| Byte[0] | 控制模式 | Uint8 | 0x01 = CAN 协议控制 |
| Byte[1-7]| 保留 | - | 填充0 |
*注：开机未连接遥控器时默认为CAN控制。遥控器具有最高权限。*

### 3.2 0x401 - 机器人运动模式下发
| 字节位置 | 字段内容 | 数据类型 | 说明 |
|---------|------------------------|---------|------|
| Byte[0] | 运动模式 | Uint8 | 0x01反向, 0x02正向(默认), 0x04自旋, 0x08自主控制, 0x10驻车 |
| Byte[1-7]| 保留 | - | 填充0 |

### 3.3 0x402 - 机器人控制状态下发 (核心运动指令) ⭐
| 字节位置 | 字段内容 | 数据类型 | 说明 |
|---------|------------------------|---------|------|
| Byte[0-1] | Vx (线速度方向)     | Int16 | 大端，数值 = 目标速度 × 1000，单位 0.001 m/s |
| Byte[2-3] | Vy (线速度方向)     | Int16 | 大端，数值 = 目标速度 × 1000，单位 0.001 m/s |
| Byte[4-5] | Vz (旋转角速度)     | Int16 | 大端，数值 = 目标角速度 × 1000，单位 0.001 rad/s |
| Byte[6-7] | 保留                | - | 0x00 |
*⚠️注意：Vz (角速度) 的编码比例系数同样是 **1000**！*

### 3.4 0x403 - 独立轮速控制 & 3.5 0x404 - 独立轮角控制
- **0x403**: 下发 `1~4` 号电机的目标转速 (各占两字节 Int16，大端，单位 0.001m/s)。顺序为：1左前、2左后、3右前、4右后。
- **0x404**: 下发 `5~8` 号电机的目标转角 (各占两字节 Int16，大端，单位 0.01°)。顺序为：5左前转向、6左后转向、7右前转向、8右后转向。

### 3.6 0x405 - 定制化外设接口输出（选配）
Byte[0] 控制 Bit0 DO1，Bit1 DO2...以此类推

## 4. 关键踩坑提示分析与订正说明（必读！）
1. **0x100 并非速度回馈**：旧版文档错误地将 0x100 记录为 Vx/Vy/Vz 返回，这完全是错误的。0x100 是底盘自身的电量/电压/电流及急停状态字典。
2. **所有报文必须采用大端序（Big-Endian）**：数据高八位分配在数组的低索引位。STM32 代码在解析时切记使用 `(data[0]<<8) | data[1]`。
3. **轮序号并非循规蹈矩的 FL/FR/BL/BR**：
   官方约定顺序：
   - 1号/5号 = 左前 (FL)
   - 2号/6号 = 左后 (RL)  <- *注意第2号和第6号电机是左后！*
   - 3号/7号 = 右前 (FR)
   - 4号/8号 = 右后 (RR)
4. **0x402 中 Vz 的乘数是 1000**：浮点转 Int16 的时候乘数也是 1000，与线速度一致！而不是此前流传的 4000！
